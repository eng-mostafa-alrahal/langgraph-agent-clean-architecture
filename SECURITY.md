# Security Policy

## Supported versions

Security updates are provided on a best-effort basis for the latest code on `main`.

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, report privately to the project maintainer with:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation if available

The maintainer will acknowledge receipt as soon as possible and work on a fix.

## Disclosure policy

- We aim to validate reports quickly.
- A fix will be prepared and released as soon as practical.
- Public disclosure should happen after a patch or mitigation is available.
